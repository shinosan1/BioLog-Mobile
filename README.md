# BioLog Mobile

BioLog Mobile は、スマホで使うことを前提にしたオフライン対応の健康記録PWAです。
サーバー、Docker、アプリストア申請なしで利用できます。ブラウザで開き、ホーム画面に追加するとアプリのように使えます。

## 公開状況

このリポジトリは、GitHub 上で public 公開してテスト利用する段階のものです。
現時点では正式版ではなく、動作確認、使い勝手の確認、バックアップ運用の確認を目的としています。

利用前、機種変更前、ブラウザのサイトデータ削除前には、必ず JSON バックアップを書き出してください。
健康記録やバックアップJSON、個人データを含むCSVは、GitHubなどの公開場所へアップロードしないでください。

## PWAについて

BioLog Mobile は PWA、Progressive Web App として作られています。
PWA はWebアプリですが、スマホのホーム画面にアイコンを追加して、通常のアプリに近い感覚で起動できます。

App Store や Google Play からインストールするアプリではありません。
公開URLをブラウザで開き、ホーム画面に追加して使います。

一度読み込んだ後は、Service Worker によりアプリ本体の画面をオフラインでも開ける場合があります。
記録データは端末のブラウザ内 IndexedDB に保存されます。

ブラウザのサイトデータを削除すると、保存した記録も削除されます。
大切な記録は、定期的に JSON バックアップを書き出してください。

## 主な機能

- 今日の健康記録入力
- 日付を指定した新規登録、編集
- 履歴の確認、編集、削除
- トップ画面の当日主要指標サマリー
- 体重、体温、血圧、脈拍、体脂肪率、基礎代謝量、筋肉量のグラフ表示
- IndexedDB による端末内保存
- JSON バックアップ書き出し、読み込み
- UTF-8 CSV 読み込み
- ライトモード / ダークモード
- PWA によるホーム画面追加とオフライン起動

## 保存されるデータ

記録は利用中の端末のブラウザ内 IndexedDB に保存されます。
外部サーバーには送信されません。

トップ画面には、当日の体重、体温、血圧、脈拍だけを表示します。
当日の記録がない場合や未入力の項目は `--` と表示され、過去データでは補完しません。

ブラウザのサイトデータを削除した場合、記録も削除されます。
機種変更、ブラウザ変更、端末初期化の前には、必ず JSON バックアップを書き出してください。

## 使い始め方

### ローカルで確認する場合

```powershell
cd path\to\biolog_mobile
python -m http.server 8787
```

ブラウザで以下を開きます。

```text
http://localhost:8787
```

### GitHub Pages で公開する場合

`biolog_mobile` の中身を GitHub Pages で配信します。
このアプリは相対パス前提で作られているため、リポジトリ名付きURLでも動作します。

公開対象に含める主なファイル:

- `index.html`
- `styles.css`
- `db.js`
- `form.js`
- `backup.js`
- `charts.js`
- `csv.js`
- `app.js`
- `manifest.webmanifest`
- `service-worker.js`
- `icons/icon-192.png`
- `icons/icon-512.png`

バックアップJSONや個人データを含むCSVは公開しないでください。

## スマホでホーム画面に追加する

### Android / Chrome

1. 公開URLを Chrome で開く。
2. メニューから「ホーム画面に追加」または「アプリをインストール」を選ぶ。
3. ホーム画面の BioLog Mobile アイコンから起動する。

### iPhone / Safari

1. 公開URLを Safari で開く。
2. 共有ボタンを押す。
3. 「ホーム画面に追加」を選ぶ。
4. ホーム画面の BioLog Mobile アイコンから起動する。

## バックアップ

バックアップタブから JSON を書き出せます。

ファイル名は以下の形式です。

```text
biolog-mobile-backup-YYYY-MM-DD.json
```

JSON は復元用の正式バックアップ形式です。
CSV 読み込みは外部データ取り込み用であり、完全な復元用途には JSON バックアップを使ってください。

## CSV 読み込み

バックアップタブから UTF-8 CSV を読み込めます。

対応カラム:

- `date`
- `weight`
- `temperature`
- `systolic_bp`
- `diastolic_bp`
- `pulse`
- `body_fat`
- `bmr`
- `muscle_mass`
- `meal_detail`
- `activity_log`
- `memo`

日本語カラムにも対応しています。

- `日付`
- `対象日`
- `体重`
- `体重(kg)`
- `体温`
- `体温(℃)`
- `収縮期血圧`
- `拡張期血圧`
- `脈拍`
- `脈拍(bpm)`
- `体脂肪率`
- `体脂肪率(%)`
- `基礎代謝`
- `基礎代謝量`
- `基礎代謝(kcal)`
- `筋肉量`
- `筋肉量(kg)`
- `食事ログ`
- `行動ログ`
- `メモ`

既存BioLogの一覧タブから書き出したCSVにも対応しています。
既存BioLog CSVに含まれる `id`, `ユーザー`, `記録日時` は読み込めますが、Mobile側の保存対象にはしません。

CSV の `id` カラムは読み込めますが、値は無視されます。
`user_id` と `date_user` は読み込めません。

## 注意事項

- 1日1件の記録として保存されます。
- 同じ日付を保存または読み込みした場合は、既存の記録が更新されます。
- CSV 内に同じ日付が複数ある場合、そのCSV全体を読み込みません。
- 既存BioLogからCSVを書き出す場合は、Mobileへ取り込む前に1ユーザーに絞ることをおすすめします。
- 複数ユーザーを含むCSVは、同じ日付が重複してMobile側で拒否される場合があります。
- 不正なCSVや範囲外の数値がある場合、途中まで取り込まず全体を拒否します。
- 空のCSVテキストセルでは、既存のテキスト内容は削除されません。

## アンインストール相当の操作

PWA のため、専用アンインストーラーはありません。

端末から消す場合は、ホーム画面のアイコンを削除し、必要に応じてブラウザのサイトデータを削除してください。
記録だけを消す場合は、バックアップタブの「全データ削除」を使います。

## 開発方針

- npm、React、Vue、Vite、Next.js、bundler は不使用
- 外部CDNは不使用
