# BioLog Mobile 操作マニュアル

## 免責事項

BioLog Mobile は日々の健康記録を補助するための個人向けツールです。
医療行為、診断、治療、助言を目的としたものではありません。

記録内容の判断、健康状態に関する判断、医療上の判断は、利用者自身の責任で行ってください。
体調不良、異常値、健康上の不安がある場合は、医師などの専門家に相談してください。

本アプリの利用、データの保存、バックアップ、消失、誤入力、読み込み失敗などによって生じたいかなる損害についても、作者は責任を負いません。

## PWAについて

BioLog Mobile は PWA、Progressive Web App です。
Webアプリですが、スマホのホーム画面にアイコンを追加して起動できます。

App Store や Google Play からインストールするアプリではありません。
公開URLをブラウザで開き、ホーム画面に追加して使います。

記録データは端末のブラウザ内に保存されます。
ブラウザのサイトデータを削除すると記録も消えるため、定期的に JSON バックアップを書き出してください。

## 1. 起動する

公開URLをスマホのブラウザで開きます。
ホーム画面に追加している場合は、BioLog Mobile のアイコンから起動します。

起動後に「端末内に保存できます。」と表示されれば、記録を保存できます。

## 2. ホーム画面に追加する

### Android

1. Chrome で BioLog Mobile を開きます。
2. 右上メニューを開きます。
3. 「ホーム画面に追加」または「アプリをインストール」を選びます。
4. 追加後、ホーム画面のアイコンから起動します。

### iPhone

1. Safari で BioLog Mobile を開きます。
2. 共有ボタンを押します。
3. 「ホーム画面に追加」を選びます。
4. 追加後、ホーム画面のアイコンから起動します。

## 3. 今日の記録を保存する

1. 「今日」タブを開きます。
2. 必要な項目を入力します。
3. 「今日の記録を保存」を押します。

入力できる項目:

- 体重
- 体温
- 収縮期血圧
- 拡張期血圧
- 脈拍
- 体脂肪率
- 基礎代謝量
- 筋肉量
- 食事ログ
- 行動ログ
- メモ

測定値がなくても、食事ログ、行動ログ、メモのいずれかがあれば保存できます。

## 4. 日付を指定して登録・編集する

1. 「日付編集」タブを開きます。
2. 対象日を選びます。
3. 「読み込む」を押します。
4. 既存の記録があれば内容が表示されます。
5. 記録がない日付なら、新規登録として入力できます。
6. 入力または修正後、「保存する」を押します。

日付を変更しただけでは保存できません。
別の日付に保存したい場合は、必ず「読み込む」を押してから入力してください。

数値項目を空欄にした場合、既存記録の更新ではその項目の既存値が保持されます。
テキスト項目は画面上の編集では空文字として保存できます。

## 5. サマリーを見る

ホーム上部に、当日の主要指標が表示されます。

- 表示項目は、体重、体温、血圧、脈拍です。
- 今日の記録がある場合は、今日の記録だけを表示します。
- 今日の記録がない場合は、各項目が `--` になります。
- 未入力の項目は `--` と表示されます。
- 今日の記録に未入力項目があっても、過去データでは補完しません。

「今日」タブ内には、主要指標の一覧も表示されます。

## 6. 履歴を見る

1. 「履歴」タブを開きます。
2. 保存済みの記録が日付順に表示されます。
3. 各カードで測定値、食事、行動、メモ、更新時刻を確認できます。

更新時刻は端末のローカル時刻で表示されます。

## 7. 履歴を編集する

1. 「履歴」タブを開きます。
2. 編集したい記録の「編集」を押します。
3. 内容を変更します。
4. 「更新する」を押します。

数値項目を空欄にした場合、その項目の既存値は保持されます。
テキスト項目は画面上の編集では空文字として保存できます。

## 8. 記録を削除する

1. 「履歴」タブを開きます。
2. 削除したい記録の「削除」を押します。
3. 確認ダイアログで削除を承認します。

削除した記録は元に戻せません。
不安な場合は、先に JSON バックアップを書き出してください。

## 9. グラフを見る

1. 「グラフ」タブを開きます。
2. 保存済みデータから折れ線グラフが表示されます。

表示されるグラフ:

- 体重
- 体温
- 血圧
- 脈拍
- 体脂肪率
- 基礎代謝量
- 筋肉量

血圧は、収縮期血圧と拡張期血圧を同じグラフに表示します。
データが2件未満の項目は、グラフの代わりに案内メッセージが表示されます。

## 10. JSONバックアップを書き出す

1. 「バックアップ」タブを開きます。
2. 「JSONを書き出す」を押します。
3. ダウンロードされたJSONファイルを安全な場所に保管します。

ファイル名の例:

```text
biolog-mobile-backup-2026-06-26.json
```

JSONバックアップは、復元に使う正式なバックアップ形式です。

## 11. JSONバックアップを読み込む

1. 「バックアップ」タブを開きます。
2. JSONファイルを選択します。
3. 「JSONを読み込む」を押します。

同じ日付の記録がある場合は更新されます。
読み込みに失敗した場合、途中まで取り込まれることはありません。

## 12. CSVを読み込む

1. 「バックアップ」タブを開きます。
2. CSVファイルを選択します。
3. 「CSVを読み込む」を押します。

CSVはUTF-8形式を推奨します。
1行目はヘッダー行にしてください。

CSV例:

```csv
date,weight,temperature,systolic_bp,diastolic_bp,pulse,memo
2026-06-26,62.5,36.6,120,80,72,朝に測定
2026-06-27,62.3,,,,,散歩した
```

日本語ヘッダー例:

```csv
日付,体重,体温,収縮期血圧,拡張期血圧,脈拍,メモ
2026-06-26,62.5,36.6,120,80,72,朝に測定
```

既存BioLogの一覧タブから書き出したCSVも読み込めます。

既存BioLog CSVで対応している主なヘッダー:

- `対象日`
- `体重(kg)`
- `体温(℃)`
- `脈拍(bpm)`
- `基礎代謝(kcal)`
- `体脂肪率(%)`
- `筋肉量(kg)`
- `収縮期血圧`
- `拡張期血圧`
- `メモ`
- `食事ログ`
- `行動ログ`

既存BioLog CSVに含まれる `id`, `ユーザー`, `記録日時` は読み込めますが、Mobile側の保存対象にはしません。

CSVの注意:

- `date` または `日付` は必須です。
- 既存BioLog CSVの場合は `対象日` が日付として扱われます。
- 日付は `YYYY-MM-DD` 形式にしてください。
- 同じCSV内に同じ日付が複数ある場合、そのCSVは読み込めません。
- 既存BioLogからCSVを書き出す場合は、Mobileへ取り込む前に1ユーザーに絞ることをおすすめします。
- 複数ユーザーを含むCSVは、同じ日付が重複してMobile側で拒否される場合があります。
- 未対応カラムがある場合、そのCSVは読み込めません。
- 空の数値セルは保存対象になりません。
- 空白だけのテキストセルは空扱いになります。
- CSVの空テキストセルでは、既存テキストは削除されません。

## 13. 全データを削除する

1. 「バックアップ」タブを開きます。
2. 「全データ削除」を押します。
3. 確認ダイアログで承認します。

端末内の記録がすべて削除されます。
削除前にJSONバックアップを書き出すことをおすすめします。

## 14. ライトモード / ダークモードを切り替える

画面右上の「ダーク」または「ライト」ボタンを押します。

- 現在ライトモードの場合は「ダーク」と表示されます。
- 現在ダークモードの場合は「ライト」と表示されます。

選んだテーマは端末のブラウザに保存されます。

## 15. オフラインで使う

一度オンラインで開いて準備が完了すると、PWAの画面はオフラインでも起動できます。
記録は端末内に保存されます。

ただし、初回表示や更新直後はオンライン接続が必要になることがあります。

## 16. データを移行する

別端末へ移行する場合:

1. 旧端末でJSONバックアップを書き出します。
2. JSONファイルを新端末へ移します。
3. 新端末でBioLog Mobileを開きます。
4. 「バックアップ」タブからJSONを読み込みます。

CSVではなく、JSONバックアップを使うことをおすすめします。

## 17. アプリを消す

BioLog Mobile はPWAのため、専用アンインストーラーはありません。

端末から消す場合:

1. ホーム画面のBioLog Mobileアイコンを削除します。
2. 必要に応じて、ブラウザのサイトデータを削除します。

サイトデータを削除すると、端末内の記録も消えます。

## 18. 困ったとき

### 保存できない

- ブラウザのプライベートモードで開いていないか確認します。
- 端末の空き容量を確認します。
- ページを再読み込みします。

### ホーム画面に追加できない

- AndroidはChrome、iPhoneはSafariで開いてください。
- 公開URLが正しいか確認してください。

### 読み込みに失敗する

- JSONはBioLog Mobileで書き出したバックアップを使ってください。
- CSVはUTF-8形式、1行目ヘッダーありにしてください。
- 日付形式が `YYYY-MM-DD` になっているか確認してください。

### 画面が古いままに見える

- ブラウザで再読み込みします。
- それでも変わらない場合は、ブラウザのサイトデータ削除後に開き直します。
- サイトデータを削除する前に、必要ならJSONバックアップを書き出してください。
